Software hardware optimisation

This video shows two examples of software hardware optimisation skills in two different games, the first of which "Dino Tiles" was mainly optimised by switching to a data oriented design and monitoring the code quality, a good example of this is changing data types from doubles to floats when floats are not necessary. In the 2nd game which just shows particles moving across a screen was mainly optimised using threads, although other methods I used include passing by refference and once again monitoring the code quality. I also used imGUI to monitor certain statistics

Scripting support

In university I was tasked with adding Lua Scripting support to a space invaders clone made by the lecturer. In this project I implemented new variables such as an enemy counter, a player laser sprite variable to control the player's weapons and another sprite variable for the mothership. I also added a player lives variable and variables and functions to control the player's movement and combat. I also called multiple c++ functions inside of lua one of which being a function that allowed the changing of a start message for the game.
